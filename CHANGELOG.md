# Changelog

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/).

## [Unreleased]

### Добавлено

- Модульная структура: `config_manager.py`, `sync_core.py`, `ui_main.py`, `ui_settings.py`; точка входа — `main.py`.
- Чекбоксы активности пар в окне настроек; префиксы `[ACTIVE]` / `[INACTIVE]` в `config.cfg`; неактивные пары пропускаются при синхронизации.
- Подробное логирование: каждый скопированный файл (путь, размер, дата изменения); список проверяемых/создаваемых директорий; прогресс сканирования и копирования.
- Документация по маскам исключений (`fnmatch`, в т.ч. `*_env`).

### Изменено

- **Критерий копирования:** файл копируется только если в источнике он **новее по дате изменения** (`mtime`), чем в назначении; размер не сравнивается для решения «копировать или нет».
- **Пути:** допускаются корни дисков (`R:\`, `C:\` и т.д.); убрана ошибочная проверка `len(path) <= 3`.
- **GUI:** периодические обновления во время сканирования, сверки списков, копирования и удаления (снижение «зависаний»).
- Логирование: `loguru` с `enqueue=True`, вывод INFO в консоль при запуске из терминала.
- **Скрипты запуска** (`run_sync.cmd`, `run_sync_silent.cmd`, `setup_venv.cmd`, `convert_line_endings.cmd`): используют `venv\Scripts\python.exe` напрямую вместо `activate.bat`. Это надёжнее, т.к. не зависит от PATH и работает в любых контекстах запуска.
- **README.md**: переписан для публикации на GitHub, добавлены инструкции по установке и использованию.

### Исправлено

- Ложное сообщение «Не заданы пути» для пар вида `Z:\R ===>>> R:\` (корень диска).
- **Venv**: пересоздан с текущей версией Python (был сломан из-за переезда с Python 3.10 на 3.14).
