# Анализ проекта FolderSync

**Дата анализа:** 03 июля 2026

---

## Краткое резюме

**FolderSync** — настольное приложение на Python/tkinter для автоматической синхронизации и резервного копирования папок по настраиваемому списку пар каталогов.

---

### Основные файлы:

| Файл | Назначение |

## Архитектура

### Компоненты:

1. **ConfigManager** — загружает `config.cfg`, парсит строки формата `<SRC> ===>>> <DST> | EXCLUDE: ...`, поддерживает префиксы `[ACTIVE]`/`[INACTIVE]`.

2. **sync_core** — ядро синхронизации:
   - `scan_directory()` — рекурсивное сканирование с исключениями
   - `should_copy_file()` — сравнение по `mtime`
   - `sync_pair()` — полный цикл синхронизации одной пары
   - `SyncResult` — статистика выполнения

3. **MainWindow** — главное окно tkinter:
   - Кнопка "Старт"/"Стоп"
   - Прогресс-бар
   - Текстовая область с логом

4. **SettingsWindow** — окно настроек:
   - Список пар с чекбоксами активности
   - Диалоги выбора папок
   - Поля редактирования путей и исключений

### Поток данных:

```
config.cfg → ConfigManager → MainWindow → sync_pair() → SyncResult
```

---

## Конфигурация

Формат `config.cfg`:

```text
[ACTIVE] <ИСТОЧНИК> ===>>> <НАЗНАЧЕНИЕ> | EXCLUDE: маски
[INACTIVE] <ИСТОЧНИК> ===>>> <НАЗНАЧЕНИЕ>
```

Маски исключений: `fnmatch` (`*.tmp`, `cache/*`, `__pycache__`).

## Зависимости

- **Python** 3.10+
- **tkinter** (стандартная библиотека)
- **loguru** (единственная внешняя зависимость)

## Установка и запуск

```bash
git clone https://github.com/kabzon93region/bkp_copy.git
cd bkp_copy
setup_venv.cmd          # создание venv + установка loguru
run_sync.cmd            # запуск с консолью
```

Скрипты используют `venv\Scripts\python.exe` напрямую без активации через `activate.bat`.
