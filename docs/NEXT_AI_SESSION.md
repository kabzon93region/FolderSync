# Инструкция для следующего чата разработки (ИИ)

Этот файл помогает **продолжить работу над проектом в новом чате** без потери контекста. Прочитай его в начале сессии.

---

## 1. Что это за проект

**FolderSync** — настольное приложение на Python **tkinter** для резервного копирования / синхронизации папок по списку пар из `config.cfg`.

- **Язык интерфейса и логов:** русский.
- **ОС:** Windows (пути, запуск).
- **Зависимость:** `loguru` (остальное — стандартная библиотека).

## 3. Ключевая логика (не ломать без согласования)

### Формат `config.cfg`

- Строка: `<SOURCE> ===>>> <DEST>` или с `| EXCLUDE: маски` (через `;`).
- Опционально префикс **`[ACTIVE]`** или **`[INACTIVE]`** — неактивные пары не синхронизируются, но остаются в списке.
- Без префикса пара считается активной (обратная совместимость).

### Копирование

- **`should_copy_file`:** копировать, если файла нет в назначении **или** дата изменения источника **строго новее** назначения (`mtime`). Размер **не** используется для этого решения.

### Исключения

- В `scan_directory` каталоги из масок вырезаются из `os.walk` (`dirs.remove`), файлы не попадают в словарь — исключения применяются **на этапе сканирования**.

### Пути

- Допустимы **корни дисков** (`R:\`, `Z:\` и т.д.). Проверка «путь не задан» — не по длине `<= 3`, а по пустоте / минимальной длине после `strip()` (см. `sync_pair` в `sync_core.py`).

### Логи

- Файл: `logs/log.log`. В консоль при запуске из терминала идёт INFO.
- Каждый успешно скопированный файл логируется с путём, размером и датой изменения (см. цикл копирования в `sync_pair`).

## 5. Как продолжить работу в новом чате

1. Указать модель: продолжение разработки **FolderSync**, репозиторий `bkp_copy`.
2. Попросить прочитать: **`docs/NEXT_AI_SESSION.md`**, **`CHANGELOG.md`**, при смене логики — **`sync_core.py`** и **`config_manager.py`**.
3. Соблюдать **`RULES.md`** и не менять формат `config.cfg` без обратной совместимости и обновления `docs/configuration.md` + README.
