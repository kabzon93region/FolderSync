## Обзор и архитектура FolderSync

FolderSync — это настольное приложение на Python с графическим интерфейсом (`tkinter`) для синхронизации содержимого папок по списку пар каталогов, заданных в файле `config.cfg`.

---

### Основные компоненты

1. **Точка входа (`main.py`)**:
   - инициализация логирования (`loguru`);
   - создание экземпляра `ConfigManager`;
   - запуск главного окна приложения.

2. **Модуль конфигурации (`config_manager.py`)**:
   - класс `ConfigManager`:
     - загрузка и сохранение `config.cfg`;
     - управление списком пар папок;
   - функция `parse_config_line()`:
     - парсинг строк конфигурации с поддержкой активности и исключений;
     - возвращает структуру с путями, масками исключений и флагом активности.

3. **Ядро синхронизации (`sync_core.py`)**:
   - функции работы с файловой системой:
     - `scan_directory()` — рекурсивное сканирование директории;
     - `is_excluded()` — проверка файла/папки на соответствие маскам исключений;
     - `should_copy_file()` — определение необходимости копирования: файл копируется, если его нет в назначении **или** дата изменения в источнике **новее**, чем в назначении (сравнение по `mtime`; размер для этого решения не используется);
   - функция `sync_pair()`:
     - выполняет синхронизацию одной пары папок;
     - использует callback-функции для обновления прогресса;
     - возвращает объект `SyncResult` с результатами синхронизации.

4. **Главное окно (`ui_main.py`)**:
   - класс `MainWindow`:
     - создание и управление главным интерфейсом;
     - обработка запуска/остановки синхронизации;
     - отображение прогресса и логов в текстовой области;
     - координация работы с `ConfigManager` и `sync_core`.

5. **Окно настроек (`ui_settings.py`)**:
   - класс `SettingsWindow`:
     - отдельное окно (`tk.Toplevel`) для редактирования конфигурации;
     - редактирование списка пар папок с чекбоксами активности;
     - выбор папок через диалоги;
     - сохранение изменений через `ConfigManager`.

6. **Логирование (`loguru`)**:
   - настраивается в `main.py` при старте приложения;
   - пишет сообщения в файл `logs/log.log` с ротацией и сжатием;
   - используется во всех модулях для:
     - информационных сообщений;
     - отладочных записей (пути, параметры файлов);
     - фиксации ошибок и исключений.

---

### Поток работы (high-level)

1. Пользователь запускает `main.py`.
2. Программа:
   - настраивает логирование (`loguru`);
   - создаёт экземпляр `ConfigManager` и загружает `config.cfg` (или создаёт его, если отсутствует);
   - создаёт и запускает экземпляр `MainWindow`.
3. Пользователь, при необходимости, открывает окно **«Настройки»** (`SettingsWindow`):
   - добавляет/редактирует пары `"источник ===>>> приёмник"` с чекбоксами активности;
   - сохраняет изменения через `ConfigManager.save_config()`.
4. Пользователь нажимает **«Старт»**:
   - `MainWindow` получает список пар из `ConfigManager`;
   - для каждой активной пары:
     - вызывается `sync_pair()` из `sync_core`;
     - через callback-функции обновляются прогрессбар и текстовая область;
     - результаты синхронизации логируются.
5. По завершении всех задач:
   - выводится итоговое сообщение о времени синхронизации;
   - приложение остаётся открытым, пока пользователь не закроет окно.

---

### Точки расширения

- **Дополнительные режимы синхронизации**:
  - поддержка «сухого прогона» (dry run), когда `sync_pair()` только рассчитывает и логирует план действий, не изменяя файловую систему;
  - новые стратегии синхронизации (зеркалирование, однонаправленное копирование и т.п.);
  - можно расширить класс `SyncResult` для более детальной статистики.

- **Тестирование**:
  - модульная структура упрощает написание автоматических тестов:
    - `sync_core.py` можно тестировать без GUI;
    - `config_manager.py` можно тестировать независимо;
    - `ui_main.py` и `ui_settings.py` можно тестировать с моками.

- **Улучшения архитектуры**:
  - можно добавить интерфейсы/абстракции для более гибкой работы с различными типами хранилищ;
  - можно добавить поддержку плагинов для расширения функциональности.
