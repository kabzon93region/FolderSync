## Руководство по использованию FolderSync

Данный документ описывает типичные сценарии работы с приложением.

---

### 1. Подготовка

1. Убедитесь, что установлен Python 3.10+.
2. Клонируйте репозиторий и перейдите в папку проекта:

```bash
git clone https://github.com/kabzon93region/bkp_copy.git
cd bkp_copy
```

3. Запустите `setup_venv.cmd` — он создаст виртуальное окружение и установит зависимости.
4. Запустите приложение через `run_sync.cmd` (с консолью) или `run_sync_silent.cmd` (без консоли).

При первом запуске:
- если `config.cfg` отсутствует, он будет создан автоматически;
- список задач синхронизации пока будет пуст.

---

### 2. Настройка списка папок для синхронизации

1. В главном окне нажмите кнопку **«Настройки»**.
2. В открывшемся окне:
   - воспользуйтесь кнопкой **«Выбрать исходную папку»** и выберите каталог-источник;
   - воспользуйтесь кнопкой **«Выбрать конечную папку»** и выберите каталог-назначение;
   - при необходимости отредактируйте пути вручную в текстовых полях;
   - при необходимости заполните поле **масок исключений** (например: `*.tmp;*.log;__pycache__`).
3. Нажмите кнопку **«Сохранить»**, чтобы:
   - добавить новую пару в список;
   - либо заменить выбранную (кликните по тексту строки в списке для выбора).
4. При необходимости:
   - **кликните по тексту строки** в списке пар, чтобы загрузить её в поля редактирования;
   - используйте кнопку **«Редактировать»** для загрузки выбранной пары (альтернатива клику);
   - используйте кнопку **«Удалить»**, чтобы удалить выбранную пару из списка.
5. **Управление активностью пар:**
   - каждая строка в списке имеет **чекбокс активности** слева от текста;
   - **установленный чекбокс** — пара будет синхронизироваться при запуске;
   - **снятый чекбокс** — пара будет пропущена при синхронизации (но останется в списке);
   - состояние чекбоксов можно изменять в любое время, кликая по ним.
6. После окончания редактирования нажмите **«OK»**:
   - изменения будут сохранены в `config.cfg` (включая состояние активности всех пар);
   - окно настроек закроется.

---

### 3. Запуск синхронизации

1. Убедитесь, что в `config.cfg` есть хотя бы одна пара папок.
2. В главном окне:
   - по желанию включите/выключите флажок **«Удалять синхронизованные файлы»**;
   - нажмите кнопку **«Старт»**.
3. Приложение по очереди выполнит синхронизацию для всех **активных** пар из списка:
   - **неактивные пары** (с снятыми чекбоксами) будут пропущены с сообщением в текстовой области;
   - в текстовой области будут появляться сообщения о текущих действиях (создание папок, копирование файлов, удаление и т.п.);
   - индикатор прогресса будет обновляться по мере выполнения.
4. После завершения всех задач в текстовой области появится сообщение о том, за сколько секунд завершилась синхронизация.

> **Важно:** 
> - если пути к папкам заданы некорректно (слишком короткие, недоступные и т.п.), в текстовой области и в лог-файле появятся соответствующие сообщения об ошибке;
> - неактивные пары не удаляются из списка, а только пропускаются при синхронизации — это позволяет временно отключать отдельные задачи без их удаления.

#### 3.1. Когда файл считается нуждающимся в копировании

Файл копируется, если в назначении его **нет** **или** дата изменения в **источнике новее**, чем в назначении (сравнение по времени изменения файла, `mtime`). Размер файла для этого решения **не** используется. Подробнее см. раздел «Критерий копировать файл или нет» в `configuration.md`.

---

### 4. Поведение при включённом удалении файлов

Флажок **«Удалять синхронизованные файлы»** управляет логикой работы после копирования:

- Если опция **включена**:
  - после успешного копирования файла из исходной папки в целевую:
    - исходный файл удаляется;
  - если файл в исходной папке не больше и не новее целевого (и при этом включён режим удаления):
    - исходный файл может быть удалён как «не требующий копирования»;
  - при невозможности удалить файл (нет прав и т.д.) в текстовой области и логе появится сообщение об ошибке.

- Если опция **выключена**:
  - никакие файлы из исходной папки не удаляются;
  - приложение работает в режиме «копирования/обновления» целевой папки.

Используйте режим удаления **только после того, как убедились**, что синхронизация выполняется корректно и указаны нужные пути.

---

### 5. Просмотр логов

Все действия приложения подробно фиксируются в файле:

- `logs/log.log`

В лог попадают:
- время запуска/остановки приложения;
- успешные и неуспешные операции копирования/удаления;
- создание директорий в целевой папке;
- диагностические сообщения при ошибках.

Лог удобно использовать для:
- разбора проблем с доступом к файлам/папкам;
- аудита выполненных операций;
- анализа времени работы на больших объёмах данных.

---

### 6. Типичные сценарии

#### 6.1. Резервное копирование загрузок на другой диск

1. Откройте **«Настройки»**.
2. Добавьте пару:

```text
D:\Downloads ===>>> E:\Archive\Downloads
```

3. Убедитесь, что флажок **«Удалять синхронизованные файлы»** **выключен**.
4. Нажмите **«Старт»**.

Результат: все файлы из `D:\Downloads` будут скопированы (или обновлены) в `E:\Archive\Downloads`, исходники останутся на месте.

#### 6.2. Перенос старых документов с системного диска

1. В окне **«Настройки»** добавьте:

```text
C:\Users\User\Documents\Old ===>>> D:\Archive\OldDocs
```

2. В главном окне включите флажок **«Удалять синхронизованные файлы»**.
3. Нажмите **«Старт»**.

Результат: документы будут перенесены в архивную папку, а исходные файлы при успешном копировании будут удалены из `Old`.

#### 6.3. Синхронизация документов с исключением временных файлов

1. В окне **«Настройки»** добавьте:

```text
C:\Users\User\Documents ===>>> D:\Backups\Docs
```

2. В поле масок исключений укажите:

```text
*.tmp;*.log;__pycache__
```

3. Нажмите **«Сохранить»**, затем **«OK»**.
4. В главном окне нажмите **«Старт»**.

Результат: обычные документы будут синхронизированы, а временные файлы (`*.tmp`, `*.log`) и каталоги `__pycache__` будут пропущены.
